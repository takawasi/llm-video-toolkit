## 1. 学術研究・論文

### LAVE (LLM-Powered Agent Assistance and Language Augmentation for Video Editing)

| 項目 | 内容 |
|------|------|
| **発表** | ACM IUI 2024 (Intelligent User Interfaces) |
| **機関** | Meta Reality Labs / University of Toronto / UCSD |
| **論文** | https://arxiv.org/abs/2402.10294 |
| **プロジェクト** | https://www.dgp.toronto.edu/~bryanw/lave/ |

**アーキテクチャ**:
```
1. 自動言語記述生成
   - 動画フッテージに対して自動で言語説明を生成
   - LLMが動画を処理するための基盤

2. Plan-and-Execute Agent
   - ユーザーコマンド受信
   - LLMが「action plan」を生成
   - ユーザー承認後にPython関数として実行

3. ハイブリッド編集
   - LLMエージェントによる自動編集
   - 直接UI操作も可能（柔軟性）
```

**機能**:
- 映像レビュー（テーマ別自動分類）
- 創意的ブレインストーミング
- 言語クエリによるクリップ検索
- ストーリーボード作成
- 意味的/時間的クリップトリミング

**使用技術**: GPT-4

**評価**: 8名のユーザー研究で有効性実証（初心者〜熟練編集者）

## 2. GitHub実装プロジェクト

### ai-video-editor (mitch7w)

| 項目 | 内容 |
|------|------|
| **URL** | https://github.com/mitch7w/ai-video-editor |
| **スター** | 44 |
| **言語** | Python 100% |
| **LLM** | Claude 3.5 Sonnet（GPT-4oより優秀との評価） |

**動作原理**:
```
1. 複数の入力動画をファイルで指定（videos.txt）
2. LLMが動画内の対話を分析
3. 「粗編集」段階で不要な部分をカット
4. 複数動画を結合して ai_output.mp4 生成
```

**制限**:
- 対話がある動画のみ対応
- 完全ではない（沈黙や誤りが残存）
- 手動後処理が必要

**ファイル構成**:
- `ai_video_editor_simple.py` - メインスクリプト
- `requirements.txt` - 依存パッケージ
- `videos.txt` - 入力ファイルリスト

**セットアップ**: .envに `OPENAI_API_KEY` と `ANTHROPIC_API_KEY` 設定

## 4. アーキテクチャパターン

### パターンA: コマンド生成型

```
ユーザー指示（自然言語）
    ↓
LLM（Claude/GPT-4）
    ↓
FFmpegコマンド or Pythonスクリプト
    ↓
実行
    ↓
出力動画
```

**実装例**: llmpeg, wtffmpeg, ffmperative-7b

**メリット**: シンプル、既存ツール活用
**デメリット**: 複雑な編集は難しい

---

### パターンB: エージェント型

```
ユーザー目標
    ↓
LLM Agent（Plan-and-Execute）
    ↓
Action Plan生成
    ↓
ユーザー承認
    ↓
Python関数/API実行
    ↓
出力動画
```

**実装例**: LAVE, ai-video-editor

**メリット**: 複雑な編集対応、対話的
**デメリット**: 実装複雑、遅い

---

### パターンC: コード生成型

```
ユーザー指示
    ↓
LLM
    ↓
Remotion/moviepy コード生成
    ↓
レンダリング
    ↓
出力動画
```

**実装例**: Remotion + GPT-4, Claude Code + Remotion

**メリット**: 高度なカスタマイズ、データ駆動
**デメリット**: 技術的ハードル高い

## 6. CB向け推奨アプローチ

### Phase 1: FFmpegコマンド生成（最小実装）

```
目標: 自然言語 → FFmpegコマンド生成 → 実行

参考:
- llmpeg（構造参考）

実装:
1. プリセット定義（カット、テロップ、音声等）

```

目標: 対話的な編集支援

参考:
